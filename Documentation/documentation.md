# 1. Въведение

Настоящият курсов проект представя разработката на конзолно приложение за онлайн магазин с функционалност за пазарска количка. Приложението е изградено на базата на C# и Entity Framework Core, като използва SQLite за съхранение на данните.

Проектът демонстрира основните принципи на обектно-ориентираното програмиране, работа с бази данни чрез ORM (Object-Relational Mapping) и създаване на многопластови софтуерни архитектури.

---

# 2. Цел на проекта

Основната цел на проекта е да се разработи функционално приложение за онлайн магазин, което позволява на потребителите да:

- Разглеждат налични продукти
- Добавят продукти в пазарска количка
- Променят количество на продукти в количката
- Премахват продукти от количката
- Преглеждат общата стойност на поръчката
- Управляват статуса на поръчките

Втората цел е да се демонстрират умения за работа с:

- Entity Framework Core Code First подход
- SQLite база данни
- NUnit тестване
- Git version control
- Многослойна архитектура на софтуера

# 4. Описание на базата данни

Базата данни се състои от три основни таблици, които моделират основните обекти в системата за онлайн магазин.

## Таблица: Product (Продукти)

Таблицата съдържа информация за всички налични продукти в магазина.

| Поле | Тип | Описание |
|------|-----|----------|
| ProductId | int (PK) | Уникален идентификатор |
| Name | string | Име на продукта |
| Description | string | Описание на продукта |
| Price | decimal | Цена на продукта |
| StockQuantity | int | Наличност в склада |
| Category | string | Категория на продукта |

**Примерни записи:**

| ProductId | Name | Price | StockQuantity | Category |
|-----------|------|-------|---------------|----------|
| 1 | Мишка | 25.99 | 50 | Периферия |
| 2 | Клавиатура | 70.00 | 30 | Периферия |
| 3 | Монитор | 350.00 | 15 | Дисплеи |
| 4 | Слушалки | 80.00 | 40 | Аудио |
| 5 | Лаптоп | 1500.00 | 10 | Компютри |

## Таблица: Cart (Количка)

Таблицата съдържа информация за всяка създадена пазарска количка/поръчка.

| Поле | Тип | Описание |
|------|-----|----------|
| CartId | int (PK) | Уникален идентификатор |
| CustomerName | string | Име на клиента |
| DateCreated | DateTime | Дата на създаване |
| TotalPrice | decimal | Обща стойност |
| Status | CartStatus (enum) | Статус: Active, Ordered, Shipped |
| Address | string | Адрес за доставка |

**Примерни записи:**

| CartId | CustomerName | DateCreated | TotalPrice | Status | Address |
|--------|--------------|-------------|------------|--------|---------|
| 1 | Иван Петров | 2026-06-01 | 95.99 | Active | ул. Главна 1, София |
| 2 | Мария Иванова | 2026-06-05 | 350.00 | Ordered | бул. България 25, Пловдив |
| 3 | Георги Стоянов | 2026-06-10 | 1580.00 | Shipped | ул. Вардар 8, Варна |
| 4 | Елена Димитрова | 2026-06-15 | 80.00 | Active | ж.к. Тракия 45, Бургас |
| 5 | Николай Христов | 2026-06-18 | 1575.99 | Active | ул. Пирин 12, Русе |

## Таблица: CartItem (Елементи в количката)

Таблицата свързва продуктите с количката и съдържа информация за конкретните артикули във всяка количка.

| Поле | Тип | Описание |
|------|-----|----------|
| CartItemId | int (PK) | Уникален идентификатор |
| CartId | int (FK) | Връзка към Cart |
| ProductId | int (FK) | Връзка към Product |
| Quantity | int | Количество |
| UnitPrice | decimal | Цена на единица |
| AddedDate | DateTime | Дата на добавяне |

## Връзки между таблиците

- **Cart (1) ---→ (N) CartItem**: Една количка може да съдържа много елементи
- **Product (1) ---→ (N) CartItem**: Един продукт може да присъства в много колички

Тези връзки се реализират чрез външни ключове (Foreign Keys) в таблицата CartItem.

# 5. Реализация на проекта

## Структура на решението

Проектът е организиран в многопластова архитектура с четири отделни проекта:

```
Viktor_Nikolchev_11J/
├── Viktor_Nikolchev_11J.slnx
├── ShoppingCart.Models/        # Модели на данните
│   ├── Product.cs
│   ├── Cart.cs
│   ├── CartItem.cs
│   └── CartStatus.cs
├── ShoppingCart.Data/          # Достъп до данни (DbContext)
│   └── ShoppingCartContext.cs
├── ShoppingCart.Services/      # Бизнес логика
│   ├── ProductService.cs
│   └── CartService.cs
├── ShoppingCart.ConsoleApp/    # Конзолен интерфейс
│   └── Program.cs
└── ShoppingCart.Tests/         # Тестове
    └── ProductTests.cs
```

## Слой Models

Моделите дефинират структурата на данните. Всеки модел съответства на таблица в базата данни.

**Product.cs** съдържа свойствата Id, Name, Description, Price, StockQuantity и Category, както и навигационно свойство към CartItems.

**Cart.cs** съдържа информация за клиента, дата, обща цена, статус и адрес.

**CartItem.cs** свързва Cart и Product чрез външни ключове.

## Слой Data (DbContext)

ShoppingCartContext наследява DbContext и управлява връзката с базата данни. Конфигурацията включва:

- Дефиниране на DbSet за всяка таблица
- Конфигуриране на връзките между таблиците (Fluent API)
- Seed data с примерни записи
- Конвертиране на enum CartStatus към string в базата данни

## Слой Services (Бизнес логика)

**ProductService** предоставя CRUD операции за продукти:

- Create - добавяне на нов продукт
- Read - преглед на всички продукти или търсене по ID
- Update - промяна на цена или количество
- Delete - изтриване на продукт

**CartService** предоставя операции за количка:

- CreateCart - създаване на нова количка
- GetCart - преглед на количка с всички елементи
- AddToCart - добавяне на продукт в количка
- UpdateStatus - промяна на статус (Active/Ordered/Shipped)
- DeleteCart - изтриване на количка
- CalculateTotal - изчисляване на обща цена

## Конзолно приложение

Приложението предоставя интерактивно текстово меню:

```

2. Добави продукт
3. Редактирай продукт
4. Изтрий продукт
5. Създай количка
6. Добави продукт в количка
7. Покажи количката
8. Промени статус на количка
9. Изтрий количка
10. Изчисли обща цена
0. Изход
```

## Тестове

Използвани са NUnit 4.3.2 и Entity Framework InMemory за тестване на CRUD операциите:

- **AddProduct_ShouldIncreaseProductCount** - тест за добавяне на продукт
- **GetProductById_ShouldReturnCorrectProduct** - тест за търсене на продукт
- **UpdateProduct_ShouldChangePrice** - тест за промяна на цена
- **DeleteProduct_ShouldRemoveProduct** - тест за изтриване на продукт
- **GetAllProducts_ShouldReturnAllProducts** - тест за всички продукти
- **GetProductById_NonExistent_ShouldReturnNull** - тест за несъществуващ продукт

## Инсталиране и стартиране

За да стартирате проекта:

1. Клонирайте хранилището
2. Отворете терминал в главната директория
3. Изпълнете `dotnet restore`
4. Изпълнете `dotnet run --project ShoppingCart.ConsoleApp`
5. За тестовете: `dotnet test`
